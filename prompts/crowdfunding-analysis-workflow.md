# 募资平台可行性分析工作流

## 核心流程

```
项目资料 
     ↓ 
第一步：识别项目画像 
     → 这是什么品类？ 
     → 它解决什么问题？ 
     → 它像谁？ 
     ↓ 
第二步：去募资平台验证 
     → 搜索同类项目 
     → 对比关键指标 
     → 找出成功/失败差异点 
     ↓ 
第三步：回推判断 
     → 在生态中的位置？ 
     → 机会在哪？风险在哪？ 
```

---

## 第一步：识别项目画像

### 1.1 品类识别

从项目资料中自动判断品类：

| 品类 | 特征 | 关键词 |
| --- | --- | --- |
| AI硬件 | 实体设备+AI功能 | AI、智能、硬件、设备、机器人 |
| 软件工具 | 纯软件服务 | SaaS、APP、工具、平台、软件 |
| 消费电子 | 电子消费品 | 耳机、手表、相机、音箱、台灯 |
| 生活方式 | 生活改善类 | 家居、厨房、健康、美容、旅行 |
| 文创设计 | 创意设计类 | 设计、文创、艺术、手作、文具 |
| 游戏娱乐 | 游戏娱乐类 | 游戏、玩具、桌游、卡牌 |
| 社会创新 | 公益/社会类 | 环保、公益、社会、教育 |
| B2B/企业 | 企业服务类 | 企业、商用、工业、专业 |

### 1.2 痛点分析

回答以下问题：
- 用户的核心痛点是什么？
- 痛点是否明确、具体、可感知？
- 用户当前是如何解决这个问题的？
- 现有解决方案有什么不足？
- 这个痛点有多痛？（高频刚需 vs 低频可选）

### 1.3 参照项目识别

从竞品数据库中找出最相似的3-5个项目：
- 相同或相似品类
- 解决相同或相似问题
- 目标用户相似

---

## 第二步：去募资平台验证

### 2.1 搜索策略

使用以下关键词组合搜索：
- 品类关键词（AI硬件、智能穿戴、智能家居等）
- 功能关键词（AI、智能、语音、翻译等）
- 场景关键词（办公、家居、健康、教育等）

### 2.2 对比指标

| 指标 | 说明 | 获取方式 |
| --- | --- | --- |
| 募集金额 | 已募集金额/目标金额 | 项目页面 |
| 达成率 | 已募集金额/目标金额×100% | 计算 |
| 支持者数量 | 支持项目的人数 | 项目页面 |
| 平均支持金额 | 已募集金额/支持者数量 | 计算 |
| 价格带 | 早鸟价/标准版/豪华版价格 | 项目页面 |
| 交付周期 | 预计交付时间 | 项目页面 |
| 项目状态 | 进行中/已成功/已失败 | 项目页面 |
| 上线时间 | 项目开始时间 | 项目页面 |

### 2.3 成功/失败差异分析

分析成功项目的共性：
- 痛点是否更明确？
- 产品是否更直观？
- 价格是否更合理？
- 团队是否更可信？
- 推广是否更有效？

分析失败项目的共性：
- 痛点是否模糊？
- 产品是否过于复杂？
- 价格是否过高？
- 团队是否缺乏经验？
- 是否存在履约风险？

---

## 第三步：回推判断

### 3.1 生态位置判断

回答以下问题：
- 这个品类在募资平台上的竞争激烈程度如何？
- 市场是蓝海还是红海？
- 用户对这类产品的接受度如何？
- 是否有明显的市场缺口？

### 3.2 机会识别

识别潜在机会：
- 是否有未被满足的细分需求？
- 是否有差异化的切入点？
- 是否有更好的定价策略？
- 是否有更好的推广渠道？

### 3.3 风险识别

识别潜在风险：
- 竞争风险：是否面临激烈竞争？
- 技术风险：技术是否成熟？
- 履约风险：能否按时交付？
- 市场风险：用户是否愿意付费？
- 合规风险：是否符合平台规则？

## 竞品数据库查询指南

### 数据库位置
- 数据文件：`tools/data/competitors.json`
- 分析结果：`tools/data/competitor_analysis.json`

### 可用数据

**品类列表**：AI硬件、智能穿戴、智能家居、机器人、AI配件

**字段说明**：
- name：项目名称
- category：品类
- description：项目描述
- amount_raised：已募集金额
- target_amount：目标金额
- backers：支持者数量
- success_rate：达成率
- status：状态（已成功/进行中）
- price_range：价格区间
- launch_date：上线日期
- delivery_date：预计交付日期

### 查询示例

**按品类查询**：
```python
db.search_competitors(category="AI硬件")
```

**按关键词查询**：
```python
db.search_competitors(keyword="AI")
```

**组合查询**：
```python
db.search_competitors(keyword="语音", category="AI硬件")
```

## 使用说明

1. 用户提供项目资料
2. Agent 执行第一步：识别项目画像
3. Agent 查询竞品数据库，执行第二步：募资平台验证
4. Agent 基于验证结果，执行第三步：回推判断
5. 输出完整分析报告
