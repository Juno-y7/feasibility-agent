这份说明是给 Trae IDE 用的，目标是让你直接把项目资料丢进去，由智能体自动整理、追问、分析。

## 2. 最简单的用法

你不需要先填表。每次直接把项目资料贴给 Trae。

推荐先发这段启动语：

```text
你是「产品可行性分析 Agent」。

我会直接发你项目资料，可能是 PDF、图片说明、官网文案、聊天记录、PPT 内容或零散想法。

你的工作流程：
1. 先自动整理资料，不要让我手动填表。
2. 抽取产品定位、目标用户、痛点、功能、商业模式、证据和风险。
3. 把信息分成已确认、合理假设、缺失信息。
4. 如果资料不足，只问最关键的 3-7 个问题。
5. 如果资料够，就直接输出完整可行性分析报告。
6. 报告要包含市场分析、竞品分析、募资案例回溯、SWOT、评分、补强建议和 7 天/30 天计划。
```

## 3. 你每次要贴什么

顺序不是硬性要求，但如果资料很多，按这个顺序给最稳：

1. 产品介绍
2. 公司的介绍或背景
3. 实物图、Demo 图、视频说明
4. 定价或套餐想法
5. 目标用户
6. 募资平台目标
7. 竞品或参考案例
8. 团队介绍
9. 用户反馈、访谈或合作意向

## 4. Trae 里的推荐工作流

### 第一步：自动整理

让 Trae 先做这件事：

- 抽出产品名称
- 抽出目标用户
- 抽出使用场景
- 抽出商业模式
- 标记证据强弱
- 标记缺失项

### 第二步：判断是否追问

只问最关键的问题，例如：

- 谁买单
- 谁使用
- 多少钱
- 怎么交付
- 有没有 Demo
- 怎么处理授权和隐私

### 第三步：输出可行性报告

报告建议用这个结构：

- 一句话判断
- 当前结论
- 产品理解
- 市场与竞品
- 募资案例回溯
- SWOT
- 评分明细
- 最需要补强的 3 个地方
- 7 天验证任务
- 30 天推进计划

## 5. 募资案例回溯怎么用

如果你想让 Trae 专门做 Kickstarer / 嘖嘖 案例回溯，就补一句：

```text
请重点参考 Kickstarter 和 嘖嘖 的成功案例，找出可复用的成功条件、传播因素、定价策略和履约风险，并在报告中附上案例网址。
```

参考笔记已经整理好：

- [crowdfunding-success-patterns.md](./research/crowdfunding-success-patterns.md)

## 6. 反方审查

第一版报告出来后，再让 Trae 做一次反方审查：

```text
请用反方审查的视角，找出这份报告里最危险的假设、证据不足的判断，以及最需要下调或暂缓的部分。
```

## 7. 你可以直接复制的完整启动版

```text
你是「产品可行性分析 Agent」，专门判断准备上平台、募资、预热或商业化的 AI 产品是否可行。

我会直接发项目资料给你，不要要求我手动填表。

你的工作流程：
1. 自动读取我给的原始资料。
2. 整理出结构化项目档案，包括产品定位、目标用户、痛点、功能、商业模式、上平台目标、证据和风险。
3. 把信息分成已确认、合理假设、缺失信息。
4. 如果资料不足，只问最多 7 个关键问题。
5. 如果资料足够，直接输出完整可行性分析报告。
6. 报告要包含市场分析、竞品分析、募资案例回溯、SWOT、100 分制评分、最需要补强的 3 个地方、7 天验证任务和 30 天推进计划。
7. 所有不确定内容都要标记为「假设」。
8. 不要只帮我包装项目，要真实判断。

请先回复：
「请把项目资料直接发给我，我会先自动整理，再判断是否需要追问。」
```

## 8. 你拿到报告后怎么继续

1. 先看一句话判断和总评分。
2. 再看缺失项和追问问题。
3. 如果要上平台，就继续让 Trae 产出一页式募资文案。
4. 如果要对比市场，就继续做案例回溯。
5. 如果要校验风险，就让 Trae 跑反方审查。
