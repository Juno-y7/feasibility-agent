# 募资文案优化模板

## 标题优化

### 标题结构

```
{产品名} | {核心卖点} | {差异化}
```

### 标题公式

| 公式类型 | 示例 | 适用场景 |
| --- | --- | --- |
| 痛点解决 | "再也不用...的..." | 强痛点产品 |
| 功能描述 | "能...的..." | 功能型产品 |
| 场景描述 | "在...场景下的..." | 场景型产品 |
| 对比型 | "比...更好的..." | 有明确竞品 |
| 数字型 | "仅需X元，获得..." | 价格敏感型 |
| 情感型 | "让你...的..." | 情感型产品 |

### 标题检查清单

- [ ] 3秒内看懂产品价值
- [ ] 包含产品名和核心卖点
- [ ] 有差异化或独特性
- [ ] 口语化，容易传播
- [ ] 不超过20个字

### 首图类型

| 类型 | 适用场景 | 示例 |
| --- | --- | --- |

### 首图检查清单

- [ ] 产品清晰可见
- [ ] 核心价值一目了然
- [ ] 高质量，无模糊
- [ ] 符合平台尺寸要求
- [ ] 有视觉吸引力

## 视频脚本优化

### 视频结构

#### 0-10秒：钩子

```
- 场景：展示用户痛点场景
- 问题：提出用户面临的问题
- 悬念：留下悬念，吸引继续观看
```

#### 10-30秒：问题

```
- 放大痛点：详细描述用户痛点
- 现状：展示用户当前的解决方案
- 问题：指出当前方案的不足
```

#### 30-60秒：产品

```
- 产品亮相：展示产品
- 核心功能：演示核心功能
- 解决问题：展示产品如何解决问题
```

#### 60-90秒：演示

```
- 使用演示：真实使用场景演示
- 效果对比：前后对比或与竞品对比
- 价值展示：展示产品带来的价值
```

#### 90-120秒：行动

```
- 价值总结：总结产品核心价值
- 早鸟优惠：展示早鸟价格和优惠
- 呼吁行动：引导用户支持
```

### 视频检查清单

- [ ] 开头3秒抓住注意力
- [ ] 有明确的问题-解决方案结构
- [ ] 产品演示真实可信
- [ ] 有情感共鸣
- [ ] 有明确的行动号召

## 正文优化

### 痛点描述

#### 写法公式

```
{场景} + {问题} + {后果}
```

#### 示例

```
每天通勤路上，你是否为了找一个插座而焦虑？
手机没电时，重要的电话接不到，紧急的消息回不了，
错过的可能不只是信息，更是机会。
```

### 产品介绍

#### 写法公式

```
{产品名}是一款{品类}，它能{核心功能}，
帮助{目标用户}在{使用场景}下解决{痛点}，
带来{核心价值}。
```

### 核心功能

#### 写法公式

```
- 功能名：一句话描述功能
- 解决什么问题：这个功能解决了什么痛点
- 带来什么价值：用户能获得什么好处
```

#### 示例

```
- 超长续航：一次充电，使用7天
  - 解决频繁充电的烦恼
  - 出门在外不用担心没电
  
- 快速充电：30分钟充至80%
  - 解决紧急情况下充电慢的问题
  - 临时充电也能快速恢复
```

### 使用场景

#### 写法公式

```
- 场景名：一句话描述场景
- 具体情况：描述具体使用情境
- 产品价值：产品在这个场景下的价值
```

### 技术参数

#### 写法原则

1. **只展示关键参数**：不要罗列所有参数，只展示用户关心的
2. **用用户易懂的语言**：避免专业术语，用通俗的语言解释
3. **提供对比数据**：与竞品或传统方案对比
4. **用可视化方式**：表格、图表、对比图等

### 团队介绍

#### 写法公式

```
- 团队名：团队名称
- 核心成员：关键成员的背景和经验
- 过往成就：团队或成员的过往成功案例
- 为什么能做好：为什么这个团队能做出好产品
```

#### 要点

- 真实可信：提供真实照片和信息
- 相关经验：展示与产品相关的经验
- 能力证明：提供可验证的成就

### 早鸟优惠

#### 写法公式

```
- 超级早鸟（限量X份）：￥XXX（原价￥XXX，省XXX）
  - 包含内容
  - 限量说明
  
- 早鸟（限量X份）：￥XXX（原价￥XXX，省XXX）
  - 包含内容
  - 限量说明
  
- 标准版：￥XXX
  - 包含内容
  
- 豪华版：￥XXX
  - 包含内容
  - 增值服务
```

#### 要点

- 价格梯度清晰
- 限量制造紧迫感
- 每个档位有明确价值差异
- 显示原价和折扣，突出优惠

### 进度计划

#### 写法公式

```
| 阶段 | 时间 | 任务 |
| --- | --- | --- |
| 设计确认 | 已完成 | 最终设计和打样 |
| 供应链确认 | 已完成 | 供应商和交期确认 |
| 量产准备 | 募资中 | 模具和原材料准备 |
| 生产制造 | 募资后1-2月 | 批量生产 |
| 质量检测 | 生产完成后 | 产品检测 |
| 包装发货 | 检测通过后 | 包装和物流 |
```

#### 要点

- 时间节点明确
- 任务描述清晰
- 显示当前进度
- 给用户信心

## A/B测试建议

### 测试维度

| 测试项 | 版本A | 版本B | 测试指标 |
| --- | --- | --- | --- |
| 标题 | 痛点型 | 功能型 | 点击率 |
| 首图 | 产品图 | 场景图 | 转化率 |
| 视频 | 短版(60秒) | 长版(120秒) | 完播率 |
| 定价 | 3档位 | 4档位 | 平均支持金额 |

### 测试方法

1. 小范围测试：在社群或朋友圈测试
2. 数据收集：记录点击率、转化率等指标
3. 分析结果：对比不同版本的表现
4. 优化调整：根据测试结果优化文案

## 优化技巧

### 心理技巧

1. **稀缺原则**：限量早鸟制造紧迫感
2. **对比原则**：与竞品或现状对比，突出优势
3. **社会认同**：展示支持者数量、用户评价等
4. **互惠原则**：提供超值回报，让用户感觉划算
5. **承诺一致**：清晰的进度计划，建立信任

### 写作技巧

1. **用用户视角**：从用户角度描述问题和解决方案
2. **具体而非抽象**：用具体场景和数字，不用模糊描述
3. **有情感共鸣**：触动用户的情感，不只是理性说服
4. **简洁有力**：避免冗长，每句话都有价值
5. **口语化**：像朋友聊天一样，不用生硬的文案

### 视觉技巧

1. **高清图片**：高质量的产品图和场景图
2. **视频优先**：视频比图片更能展示产品价值
3. **对比展示**：前后对比、竞品对比
4. **信息图表**：用图表展示数据和参数
5. **一致性**：整体视觉风格统一
