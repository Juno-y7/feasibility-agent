# 网页版可行性报告模板

## 输出标准

所有产品可行性分析报告，默认输出为**静态网页格式**（单文件HTML），无需部署，可直接分享。

## 技术栈
- HTML5 + CSS3
- Tailwind CSS (CDN)
- Chart.js (CDN)
- 字体：Google Fonts - Noto Sans SC

## 报告结构（共15章）

| 章节 | 内容 |
|------|------|
| 0. 执行摘要 | 项目概述、核心结论、关键指标（评分/市场规模/毛利率/回收期） |
| 1. 项目画像识别 | 产品定位、用户分层、产品版本规划、产品图 |
| 2. 市场分析 | 市场规模趋势图、品类份额、市场趋势 |
| 3. PEST分析 | 政治/经济/社会/技术 |
| 4. 波特五力分析 | 供应商/购买者/新进入者/替代品/行业竞争 |
| 5. SWOT分析 | 优势/劣势/机会/威胁 |
| 6. 商业模式画布 | 9大模块 |
| 7. 竞品案例研究 | 真实案例、可点击链接 |
| 8. 市场规模估算 | TAM/SAM/SOM |
| 9. 财务模型分析 | 收入预测、成本结构、盈亏平衡 |
| 10. 增长飞轮分析 | AARRR漏斗、增长阶段 |
| 11. 技术架构评估 | 技术可行性、技术风险 |
| 12. 可行性评分 | 雷达图、10维度加权评分（满分100） |
| 13. 行动建议 | 7天验证任务、关键短板、30天计划 |
| 14. 沟通话术 | 投资人、募资平台、合作伙伴 |

## 评分模型（满分100分）

| 维度 | 权重 |
|------|------|
| 痛点强度 | 15 |
| 目标用户清晰度 | 10 |
| 竞争壁垒 | 10 |
| 产品可演示性 | 10 |
| 技术/生产可行性 | 10 |
| 合规与信任 | 10 |
| 商业模式 | 10 |
| 市场推广可行性 | 10 |
| 团队匹配度 | 10 |
| 证据完整度 | 5 |

**分数解释**：
- 80-100：可以进入正式开发/预热/推广阶段
- 65-79：方向可行，但必须先补强2-3个关键短板
- 50-64：需要重新定义用户、场景或卖点
- 0-49：暂不建议投入大量资源

## 设计规范

- 玻璃拟态风格（Glassmorphism）
- 深色主题（深蓝紫渐变背景）
- 主色调：蓝色(#60a5fa) + 紫色(#a78bfa)渐变
- 左侧固定导航栏（可滚动到对应章节）
- 平滑滚动 + 滚动高亮导航
- 响应式布局

## 文件位置与命名规范

- 模板文件：`template.html`
- 输出目录：`reports/{project-name}/web-report/index.html`
- 项目目录名：使用项目英文名/拼音小写，用短横线连接，如 `konglan-ai-avatar`
- 直接双击或用HTTP服务器打开即可查看

### 目录结构示例
```
feasibility-agent/
├── reports/
│   ├── konglan-ai-avatar/          # 项目目录
│   │   ├── web-report/             # 网页报告
│   │   │   └── index.html
│   │   ├── extracted-intake.md     # 结构化摘要
│   │   └── v01-feasibility.md      # Markdown版本报告
│   └── {other-project}/
│       └── web-report/
│           └── index.html
└── templates/
    └── web-report/
        ├── template.html
        └── README.md
```

## 生成步骤

1. 复制 `template.html` 到目标报告目录
2. 替换所有占位内容为实际产品信息
3. 更新所有图表数据（Chart.js配置）
4. 更新产品图片（建议用真实产品图）
5. 验证所有链接可点击
