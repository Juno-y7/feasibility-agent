# 如何直接把项目资料发给 Agent

你不需要整理成表格。可以直接这样发：

```text
我要分析一个准备上平台的项目，资料如下。请你不要让我手动填表，先自动整理项目档案，再判断是否需要追问，最后输出可行性分析报告。

【项目资料】
这里粘贴产品介绍、页面草稿、聊天记录、PPT 内容、竞品链接、导师反馈等。
```

## 推荐发法

如果资料很多，按下面顺序丢给 Agent，效果会更稳：

1. 产品介绍
2. 准备上哪个平台
3. 页面草稿或销售文案
4. 目标用户资料
5. Demo、截图、视频说明
6. 竞品或参考案例
7. 价格或回馈方案
8. 团队和交付能力
9. 导师、客户或内部讨论反馈

但这不是必填项。你只要有多少发多少，Agent 会自己整理。

## 发给 Agent 的启动句

```text
我会直接发你一个准备上平台的项目资料。请你按产品可行性分析 Agent 工作：
1. 不要让我手动填表。
2. 先自动整理项目档案。
3. 标记已确认、合理假设和缺失信息。
4. 如果必须追问，最多问 7 个关键问题。
5. 如果可以判断，就直接输出完整可行性分析报告。
```

## Agent 应该先输出什么

Agent 收到资料后，应该先输出：

```markdown
## 我从资料中整理出的项目档案

### 已确认信息
...

### 合理假设
...

### 缺失信息
...

## 是否需要追问
...

## 初步判断
...
```

然后再进入完整报告。
